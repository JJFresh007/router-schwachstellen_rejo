	\chapter*{Kurzfassung}
	Ziel der Kurzfassung ist es, einen (eiligen) Leser zu informieren, so dass
	dieser entscheiden kann, ob die Arbeit für ihn hilfreich ist oder nicht
	(neudeutsch: Management Summary).                        
	Die Kurzfassung gibt daher eine kurze Darstellung
	\begin{itemize}
		\item{des in der Arbeit angegangenen Problems}
		\item{der verwendeten Methode(n)}
		\item{des in der Arbeit erzielten Fortschritts}
	\end{itemize}
	
	Dabei sollte nicht auf die Struktur der Arbeit eingegangen werden, also Kapitel
	2 etc. denn die Kurzfassung soll ja gerade das Wichtigste der Arbeit vermitteln,
	ohne dass diese gelesen werden muss. Eine Kapitelbezogene Darstellung sollte
	sich in Kapitel 1 unter Vorgehen befinden.

	Wie der Name erahnen lässt ist der Router innerhalb eines Netzwerks dafür
	zuständig Nachrichten von einer angeschlossenen Quelle zum Ziel zu leiten.
	Der Router kümmert sich darum, dass der optimalen Weg von der Quelle zum Ziel 
	gefunden wird. Welcher Pfad gewählt wird hängt von den Konfigurationen ab,
	es kann der schnellste Weg genutzt werden oder auch eine definierte Route
	über verschiedene Punkte.
	%---Protokolle oder über was wird das festgelegt?	
	Wichtig zu Beachten ist der Unterschied zwischen Router und Switch. Beide
	Geräte sind für die Weiterleitung der Pakete an das richtige Ziel zuständig.
	Der Unterschied ist, dass ein Switch Pakete innerhalb eines Netzsegments oder
	einem lokalen Netzwerk weiterleitet, wohin gegen ein Router Pakete über 
	Netzwerkgrenzen, also auch in ein neues Netzsegment oder das Internet leiten
	kann. Switch werden daher innerhalb von lokalen Netzwerken, sofern es keine
	Segmentierung gibt oder Netzsegmenten genutzt und Router an den Grenzen der
	Netzsegmente oder vom lokalen Netzwerk in das Internet. 
	Ein Router hat heutzutage ein eingebautes Modem, das ihm ermöglicht Pakete,
	ins Internet zu senden als auch zu empfangen. Ohne ein solches Modem ist
	die Kommunikation ins Internet nicht möglich. Das Modem wandelt die digitalen
	Signale von einem Endgerät, wie einem Laptop in analoge Signale um und sendet
	dann die Anfrage in das Internet. Die Antwort ist ein analoges Signal welches
	wiederum in ein digitales gewandelt werden muss, damit der Laptop die 
	enthaltenen Informationen verarbeiten kann.
	Um zu wissen wohin ein Paket muss, untersucht der Router das angekommene Paket.
	Es enthält sowohl die Quell-, als auch die Zieladresse anhand derer der Router
	entscheiden kann, ob das Ziel eins der Geräte ist, die mit ihm verbunden sind
	oder ob das Paket weitergereicht werden muss an den nächsten Switch oder 
	Router.
	%---

	IP steht für Internet Protocol und kann folgendermaßen aussehen $198.168.1.42$

	\section{Sicherheitsmechanismen im Router}
	\subsection{Adressverbergung(NAT - RFC 30022)}
	%--------------------------------------------------------------------------------%
	%-----------Nochmal überarbeiten----formulierung und rechtschreibung anpassen----%
	Networke Address Translation(NAT) ist ein Mechanismus, bei dem der Router interne 
	IP-Adressen eines lokalen Netztwerks (LAN) in eien öffentliche IP-Adresse übersetzt.
	Es gibt welweit ca 4 Millarden public IPv4 Adressen, sie sind also begrenzt
	Private Adressen werden nur innerhalb eines lokalen Netztwerks genutzt von den Geräten.
	Haushalte oder Frimen haben alle eine private IP Adresse mit der sie mit dem Router verbunden sind.
	Der Router wiederum hat eine public IP Adresse, und hat somit zugriff auf das Internet.
	Über den Router können so alle Geräte auf das Internet zugreifen.
	NAT übersetzet also ein Satzt von IP AdrAessen zu einem anderen Satz an IP-Adressen.
	Es übersetzet also von Privat zu Public aber auch andersherum

	NAT selbst ist nicht offiziel ein Sicherheitsmechanismen, kann aber also solcher intepretiert werden
	Den Aufgrund der übersetetzung mehrerer lokaler IP-Adressen zu eine publizierten kann von
	außen nicht festgestellt werden, wie das betrachetet Netztwerk intern aufgebaut ist
	oder wie viele sich in ihm befinden. Zudem ist keine direkte verbindung von außerhalb mit
	den Geräten möglich solange dies nicht explizit erlaubt wird.

	NAT bietet eine impllizite Schutzwirkung, aber keine echte Sicherheit im klasischen Sinne.
	Nicht angeforderte verbindungen werden blockiert, die Angriffsfläche wird reduziert und Geräte können
	nicht direkt adressiert werden.

	\subsection{Paketfilterung und Stateful Firewalls}
	Eine Firewall schützt ein netztwerk vor bedrohungen, in dem sie den Datenverkehr kontrolliert.
	- zustandbehaftete(Stateful) und zustandlose(Statless) FWs
	- ersteres eher die Norm, wei besser
	Statless:
	- älter
	- verlässt dich auf Zugriffskontrolliste beim Filtern
	- entscheidung auf grundlage von Paket infos
	- speichern keine infos über status oder kontext der verbindung
	- überprüft Quell- und Ziel-IP-Adresse, Portnummer und den verkehrstyp im Header
	- Filterung nur so gut wie vom nutzer definierte -> müssen gescheit verwaltet werden
	- weniger ressourcen und schneller -> gut bei hohem Datenverkehr
	- wird heute vorallem dazu verwendet die last der staful firewall zu verringern
	- bekannte IP-Adressen blocken, geographische Regionen
	- ist nur so gut wie der Netzwerkadmin (Benutzerfehler)

	Stateful:
	- überwachen Datenpackete und den Kontext ders Datenverkehrs
	- verstehen den kontext von NW verbindungen
	- dynamische Paketflterung
	- Quell und Ziel IP Erfassung
	- können Bewegung von Daten im ganzen Netztwerk überwachen
	- durch Zustandstabelle -> entscheidung daruf basierend in der Zukunft
	- sind anfällig für DDoS und Man in the Middle, wenn Code nicht auf neustem Stand
	- auflistung der Eigenschaften

	\subsection{WLAN-Verschlüsselung}
	Bei der WLAN-verschlüsselung wird die Funkkommunikation zwischen dem Endgerät und dem 
	Access-Point abgesichert.
	- basiert auf IEEE-blabla und IEE blavla (für WPA3)
	- jedes Gerät in Reichweite kann den Datenverkehr mithören
	- durch verschlüsselung wird das verhindert
	WPA2:
	- Verschlüsselung durch AES-CCMP (Counter Mode with CBC_MAC Protokoll)
	- AES sorgt für starke symmetrische Verschlüsselung
	- CCMP = Verschlüsselung und Integritätsprüfung
	- Daten sind dann gegen Lesen und Manipulation geschützt
